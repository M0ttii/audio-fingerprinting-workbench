# Audio Fingerprinting Workbench

Research workbench for the bachelor's thesis:
**"Intelligente Audioerkennung: Analyse und Implementierung moderner Audio-Fingerprinting-Modelle im Vergleich zum Shazam-Prinzip"**

Compares three audio fingerprinting systems — Shazam (Wang, 2003), Quad (Sonnleitner & Widmer, 2016),
and NeuralFP (Chang et al., 2021) — across controlled distortion conditions on the FMA Medium
and GTZAN datasets.

## Environments

Two separate conda environments are required due to conflicting dependencies between
FAISS (Python 3.9) and the classical implementations (Python 3.10 type syntax).

| Environment | Python | Used by | Install |
|---|---|---|---|
| `audiofp_classical` | 3.10 | NB 02, NB 03, `profile_quad.py` | `conda env create -f audiofp_classical.yml` |
| `audiofp` | 3.9 | NB 00, NB 01, NB 04, NB 05, NB 06 | separate `environment.yml` in `src/pfann/` |

Activate before running any notebook or script:

```bash
conda activate audiofp_classical   # for Shazam / Quad work
conda activate audiofp             # for NeuralFP / analysis work
```

## Notebook Pipeline

Run notebooks in order. Each notebook depends on outputs from the previous one.

### NB 00 — Partitioning (`audiofp`)

Splits `fma_medium` into disjoint sets: reference database, NeuralFP training set, and OOD pool.
Produces JSON files in `data/partitions/`.

**Critical:** The three sets — training, reference DB, OOD — are strictly disjoint.
NeuralFP's training songs must not appear in the reference database (data leakage).

### NB 01 — Query Generation (`audiofp`)

Applies all distortion conditions (Groups A–D) to the reference songs and exports 10-second WAV
clips into `data/queries/dry_run/`. Writes `data/queries/manifest_dry.csv` with columns:
`track_id`, `query_path`, `ref_track_id`, `is_ood`, `group`, `condition`, `duration_sec`, `nominal_dur_s`.

Distortion groups:

| Group | Conditions | Focus |
|---|---|---|
| A — Scaling | Tempo 80–120 %, Pitch ±1–2 semitones, Speed 80–120 % (17 conditions) | Quad scale invariance |
| B — Noise / Compression | SNR 20/10/5/0/−5 dB, Room IR, MP3 128/64 kbps (8 conditions) | Shazam & NeuralFP |
| C — Real-World | Club (Tempo 110 % + SNR 5 dB + Room IR), Radio (Pitch +1 st + MP3 64 kbps + SNR 10 dB) | Combined |
| D — Query Length | 3 s / 5 s / 10 s / 15 s × Original + SNR 10 dB | Duration sensitivity |

### NB 02 — Shazam Evaluation (`audiofp_classical`)

Builds the Shazam reference index from `data/partitions/dry_ref.json`, runs all queries from
the manifest, and writes `results/dry_run/shazam_raw.csv` and `shazam_efficiency.json`.

Output columns: `system`, `track_id`, `ref_track_id`, `is_ood`, `predicted_id`, `score`,
`result_class`, `query_time_ms`, `group`, `condition`, `duration_sec`.

### NB 03 — Quad Evaluation (`audiofp_classical`)

Same pipeline for Quad. Additionally records scale estimation columns:
`detected_time_scale`, `detected_freq_scale`, `true_time_scale`, `true_freq_scale`.
Writes `results/dry_run/quad_raw.csv` and `quad_efficiency.json`.

### NB 04 — NeuralFP Training (`audiofp`)

Generates pfann track lists and config, then trains the model via subprocess call to pfann.
Checkpoint is saved in `checkpoints/dry_run/` (Dry Run) or loaded from `checkpoints/live_run/`
for Live Run analysis.

### NB 05 — NeuralFP Evaluation (`audiofp`)

Builds the FAISS reference index (`results/dry_run/pfann_db/`), runs all queries,
and writes `results/dry_run/neuralFP_raw.csv`.

### NB 06 — Cross-System Analysis (`audiofp`)

Reads all three `*_raw.csv` files and both efficiency JSONs. Computes Hit Rate, Precision,
Specificity, and scale MAE per system and condition. Writes `efficiency_table.tex` and
exports all figures to `results/plots/`.

## Utility Scripts

### `profile_quad.py` — Stage-level Quad profiler

Runs 5 queries from `manifest_dry.csv` (condition `A_original`) and measures wall time
for each pipeline stage:

| Stage | What is measured |
|---|---|
| A `load_audio` | librosa file I/O |
| B `compute_spectrogram` | STFT |
| C `extract_query_peaks` | Local-maximum filtering |
| D `build_query_quads` | Quad formation |
| E `query_radius` | Batch NN search against reference DB only |
| F `_select_and_filter_candidates` | Full filter loop (includes E) |
| G `identify` | Complete matching (F + sequence + verification) |
| H Total | A + B + C + D + G |

```bash
conda run -n audiofp_classical python profile_quad.py
```

### `reset_dry_run.py` — Dry Run reset utility

Moves all notebook-generated files to a timestamped backup directory (default) or
deletes them directly (`--delete`). Raw datasets and source code are never touched.

```bash
# Preview what would be moved (no changes):
python reset_dry_run.py --dry-run

# Move generated files to backup/YYYY-MM-DD_HHMMSS/:
python reset_dry_run.py

# Delete permanently (prompts for confirmation):
python reset_dry_run.py --delete
```

## Result Schema

All three systems produce a `*_raw.csv` with a common core schema:

| Column | Type | Description |
|---|---|---|
| `system` | str | `shazam` / `quad` / `neuralFP` |
| `track_id` | int | Query track ID |
| `ref_track_id` | float | Ground-truth reference ID (`NaN` for OOD) |
| `is_ood` | bool | True if song is not in the reference database |
| `predicted_id` | float | System's best match (`NaN` = no match) |
| `score` | float | System-specific confidence score |
| `result_class` | str | `TP` / `FP` / `FN` / `TN` |
| `query_time_ms` | float | End-to-end query latency in milliseconds |
| `group` | str | Distortion group (`A` / `B` / `C` / `D`) |
| `condition` | str | Specific condition (e.g. `A_tempo_90`, `B_snr10`) |
| `duration_sec` | float | Actual query clip duration |

Quad additionally outputs:

| Column | Type | Description |
|---|---|---|
| `detected_time_scale` | float | Estimated time-scale factor from vote histogram |
| `detected_freq_scale` | float | Estimated frequency-scale factor from vote histogram |
| `true_time_scale` | float | Ground-truth time-scale applied during distortion |
| `true_freq_scale` | float | Ground-truth frequency-scale applied during distortion |

## Metrics

Computed in NB 06 per system, group, and condition:

| Metric | Formula | Scope |
|---|---|---|
| Hit Rate | `TP / (TP + FN + FP)` | In-DB queries only |
| Precision | `TP / (TP + FP)` | All queries |
| Specificity | `TN / (TN + FP)` | OOD queries only |
| Query Time | mean / median / p95 in ms | All queries |
| Scale MAE (time) | `mean(|detected_time_scale − true_time_scale|)` | Quad, Group A |
| Scale MAE (freq) | `mean(|detected_freq_scale − true_freq_scale|)` | Quad, Group A |

Classification logic:

```python
# In-DB queries (ref_track_id is not NaN):
TP: predicted_id == ref_track_id   # correct match
FN: predicted_id is NaN            # no match returned
FP: predicted_id != ref_track_id   # wrong song returned

# OOD queries (ref_track_id is NaN):
TN: predicted_id is NaN            # correctly rejected
FP: predicted_id is not NaN        # false match
```
